● RDBMS
  - 관계형 데이터베이스 시스템
  - 테이블기반의 DBMS
    - 데이터를 테이블 단위로 관리 >> 하나의 테이블은 여러 개의 컬럼으로 구성
    - 중복 데이터를 최소화 시킴 >> 같은 데이터가 여러 컬럼 또는 테이블에 존재할 때, 데이터 수정 시 문제 발생 가능성 높아짐
    - 여러 테이블에 분산되어 있는 데이터 검색 시 테이블 간 관계(join)을 이용하여 필요한 데이터 검색

● SQL
  - DB에 있는 정보를 사용할 수 있도록 지원하는 언어
  - 대소문자 구별 x

● Database
  - 특정 기업이나 조직에서 필요한 데이터를 모아 일정한 형태로 저장해놓은 것(데이터 모음)

  1. 통합된 데이터 : 데이터의 중복을 최소화하여 데이터의 불일치 제거
  2. 저장된 데이터: 문서형태가 아니라 디스크와 같은 저장장치에 저장된 데이터
  3. 운영 데이터 : 서비스, 업무를 위해 사용되어지는 데이터(저장되어진 데이터는 조회되어 사용)
  4. 공용 데이터 : 한 사람, 하나의 업무만을 윈해 사용되어지는 것이 아닌 공동으로 사용되는 데이터

● DBMS(DataBaseManagementSystem)
  - 데이터베이스를 관리하는 프로그램, sw
  - 데이터 등로그 수정, 삭제, 조회 등의 기능들을 제공

● SQL
  - DMBS와 통신하기 위한 언어
  1. DML: 데이터 조작언어 (insert, update, delete)
  2. DQL: 데이터 질의 언어 (select)
  3. DDL: 데이터 정의 언어 = 데이터 베이스 객체들을 정의하는 언어(create,alter,drop)
  4. DCL: 데이터 제어 언어(grant, revoke)
  5. TCL: 트랜잭션 제어 언어(commit, rollback, savepoint) , TX 종료 : 모두 반영하고 종료, 모두 반양하지 않고 종료
